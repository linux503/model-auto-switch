## 🧠 智能切换算法分析

### 评分机制
算法使用多维度评分:
1. **基础评分**: 0.5 (所有模型)
2. **优先级调整**: 
   - 优先级 < 5: +0.2
   - 优先级 < 10: +0.1
3. **使用记录**: 有最后使用时间 +0.05

### 切换逻辑
1. **模型筛选**: 只考虑状态为 "active" 的模型
2. **评分排序**: 按评分从高到低排序
3. **测试验证**: 对最佳模型进行可用性测试
4. **执行切换**: 测试通过后执行切换
5. **回退机制**: 如果最佳模型测试失败，尝试次佳模型

### 实际测试中的算法表现
- **最佳模型**: deepseek/deepseek-chat (评分: 0.750)
- **评分依据**: 
  - 基础分: 0.5
  - 优先级调整: +0.2 (优先级3 < 5)
  - 使用记录: +0.05 (有最后使用时间)
- **切换决策**: 从 aicodee/MiniMax-M2.7-highspeed 切换到 deepseek/deepseek-chat
